# Práctica: Comunidades git

---

## 1. Listado de comunidades 

- **[Apache Subversion (SVN)](https://subversion.apache.org/)** — Fue muy popular antes de git. Aún lo usan algunas empresas y proyectos que prefieren un sistema centralizado de control de versiones.  
- **[Bitbucket](https://bitbucket.org/)** — Plataforma de Atlassian. Está pensada para equipos que ya usan Jira o Confluence. Fácil de usar, aunque menos popular que GitHub o GitLab.  
- **[Codeberg](https://codeberg.org/)** — Comunidad sin ánimo de lucro basada en Gitea. Su objetivo es la privacidad y el software libre, por lo que es una buena alternativa a GitHub si no quieres depender de grandes empresas.  
- **[Fossil SCM](https://fossil-scm.org/)** — Un sistema propio de control de versiones creado por el autor de SQLite. Incluye en un solo paquete repositorio, wiki y sistema de tickets.   
- **[GitLab](https://gitlab.com/)** — Similar a GitHub, pero con más opciones para organizar proyectos internos. Muy usado en empresas porque integra de forma sencilla control de versiones, gestión de tareas y automatización de pruebas (CI/CD).  
- **[Heptapod](https://heptapod.net/)** — Plataforma basada en GitLab, pero pensada para quienes prefieren usar **Mercurial (hg)** en lugar de git.  
- **[Perforce / Helix Core](https://www.perforce.com/)** — Muy usado en la industria del videojuego y en grandes compañías porque maneja bien archivos muy grandes. Es más complejo que git, pero muy potente en entornos profesionales.  
- **[SourceForge](https://sourceforge.net/)** — Una de las comunidades más antiguas. Fue muy popular antes de GitHub y aún aloja muchos proyectos open source, sobre todo software clásico.  
- **[SourceHut](https://sourcehut.org/)** — Plataforma muy minimalista y ligera. Pensada para programadores que buscan simplicidad y control total. Usa git y también integra listas de correo para colaborar.

## 3. Conclusión

Tras probar varias comunidades de control de versiones observamos lo siguiente:
 
- **GitLab** ofrece un ecosistema más completo para proyectos privados.
    
- **Bitbucket** es útil en entornos empresariales con Atlassian (Jira, Confluence).  
  
- **SourceForge** y **Codeberg** más orientado a proyectos libres y privacidad.
    
- Otros sistemas como Fossil, Mercurial o Perforce muestran que existen alternativas, aunque con comunidades más reducidas.

**Conclusión final:**  
Hoy en día casi todos usan **git** para trabajar en equipo y compartir proyectos. Entre las plataformas, **GitHub** es la más popular porque da mucha visibilidad, mientras que **GitLab** es muy completa y práctica para organizar el trabajo. Probar otras comunidades como Bitbucket o Codeberg también es útil, porque así entendemos mejor cómo funcionan y podemos elegir la que más cómoda nos resulte según el tipo de proyecto.
