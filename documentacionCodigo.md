# Documentación de Código: Python, JavaScript y PHP


| Aspecto / Lenguaje       | **Python** | **JavaScript (JSDoc)** | **PHP (PHPDoc)** |
|--------------------------|------------|-----------------------|-----------------|
| **Cómo documentar**       | Usando **docstrings** (`"""..."""`) al inicio de funciones, clases y módulos. |
| **Parámetros**            | Describir nombre, tipo y propósito con `Args:`. |
| **Valor de retorno**      | Explicar tipo y significado con `Returns:`. |
| **Errores / Excepciones** | Documentar con `Raises:` indicando qué excepciones puede lanzar. |
| **Ejemplos**              | Incluir ejemplo bajo `Example:` o en docstring. |
| **Buenas prácticas**      | - Documentar funciones, clases y módulos.<br>- Explicar propósito, parámetros, retorno y excepciones.<br>- Mantener docstrings actualizados.<br>- No explicar línea a línea, explicar **qué hace**. |
| **Control de versiones**  | Gestionar cambios con Git u otra herramienta.<br>Commits frecuentes y descriptivos.<br>Etiquetar versiones importantes con `tags`. |
| **Estructura sugerida de repositorio** | ```/python\nejercicios/\nREADME.md``` |
